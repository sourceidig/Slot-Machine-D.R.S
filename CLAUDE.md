## Project Overview

**Slot Machine D.R.S.** — Sistema de gestión de máquinas tragamonedas. Django 5 + MySQL + Tesseract OCR. Aplicación en español (Chile) para registrar lecturas de máquinas, gestionar turnos, cuadraturas de caja y recaudaciones.

## Architecture

**Single Django app** — all business logic lives in `control/`. No secondary apps.

### Models (`control/models.py` — 25 models)

Key model groups:
- **Auth:** `Usuario` (custom `AbstractUser`, field `rol` with 6 roles: admin, gerente, supervisor, tecnico, encargado, asistente)
- **Org structure:** `Sucursal` → `Zona` → `Maquina`
- **Operations:** `Turno` (shifts) → `LecturaMaquina` (readings per machine per shift)
- **Finance:** `CuadraturaCajaDiaria`, `EncuadreCajaAdmin`, `CierreTurno*`
- **Reporting:** `InformeRecaudacion*`, `ControlLecturas*`, `CicloRecaudacion`

### Views (`control/views.py` — ~4700 lines)

Monolithic views file. Contains:
- Auth: `login_view`, `logout_view`
- CRUD: Sucursales, Zonas, Máquinas, Usuarios
- Shift flow: `crear_turno`, `cierre_turno_*`
- Machine reading: `registro_lectura`, `guardar_lectura`
- Financial reconciliation: `cuadratura_caja_diaria`, `encuadre_caja_admin`
- **OCR API:** `POST /api/ocr-lectura/` — accepts image, returns `{success, entrada, salida, total}` JSON
- AJAX endpoints for dynamic filtering (machines by zone, reports, etc.)
- Excel export endpoints using openpyxl

### Access Control

Roles enforced via decorators in `control/decorators.py`:
- `@role_required('admin', 'gerente')` — restricts view to listed roles
- `@readonly_for('supervisor')` — allows read but blocks writes for role

`encargado` and `asistente` roles must select a `Sucursal` before any action (enforced by `SucursalEncargadoMiddleware`).

### OCR Flow

1. Frontend (`static/js/ocr_capturas.js`): opens camera modal via MediaDevices API, captures frame
2. POSTs image to `/api/ocr-lectura/`
3. Backend: PIL preprocessing → pytesseract (`--oem 3 --psm 6`) → parse digits
4. Returns entrada/salida/total values that auto-fill the reading form

Tesseract must be installed at the system level. On Windows the path is configured in `settings.py`.

### URL Structure (`control/urls.py`)

All URLs are under `control/` app, included from `slot_machine_drs/urls.py`. Named URLs follow the pattern `control:<name>`.

### Templates

All templates inherit from `templates/base.html`. Dynamic menu is generated by `control/menu.py` based on the user's role.

### Middleware (order matters)

1. Standard Django middleware
2. `AlertaSesionCerradaMiddleware` — alerts for session closures
3. `SucursalEncargadoMiddleware` — redirects encargado/asistente if no sucursal selected
4. `ErrorHandlerMiddleware` — captures exceptions → `logs/django_errors.log`

### Settings

- `AUTH_USER_MODEL = 'control.Usuario'`
- Language: `es-cl`, Timezone: `America/Santiago`
- Session: 8-hour max, expires on browser close
- Logging: file-based to `logs/django_errors.log`
