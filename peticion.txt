MEJORA 1 — Auto-aplicar filtros sin botón

Archivo: templates/cuadratura_diaria/list.html

Los filtros (select de sucursal, fecha desde, fecha hasta) deben aplicarse 
automáticamente al cambiar, sin necesidad de presionar "Filtrar".

Cambios:
1. Agregar id al form: <form id="filtro-form" method="get" ...>
2. Agregar ids a los inputs:
   - select sucursal → id="fil-sucursal"
   - input fecha_desde → id="fil-desde"
   - input fecha_hasta → id="fil-hasta"
3. Ocultar el botón "Filtrar" (eliminar o display:none). 
   Conservar el botón "Limpiar" con su href tal como está.
4. Agregar este script al final del bloque content:

<script>
(function () {
  const STORAGE_KEY = 'cuadratura_list_filters';
  const form = document.getElementById('filtro-form');
  if (!form) return;

  // Al cargar sin parámetros GET → restaurar desde sessionStorage
  const params = new URLSearchParams(window.location.search);
  const sinFiltros = !params.get('sucursal') && !params.get('fecha_desde') && !params.get('fecha_hasta');
  if (sinFiltros) {
    try {
      const saved = JSON.parse(sessionStorage.getItem(STORAGE_KEY) || 'null');
      if (saved && (saved.sucursal || saved.fecha_desde || saved.fecha_hasta)) {
        const newParams = new URLSearchParams();
        if (saved.sucursal)    newParams.set('sucursal', saved.sucursal);
        if (saved.fecha_desde) newParams.set('fecha_desde', saved.fecha_desde);
        if (saved.fecha_hasta) newParams.set('fecha_hasta', saved.fecha_hasta);
        window.location.replace('?' + newParams.toString());
        return;
      }
    } catch(e) {}
  }

  // Guardar en sessionStorage y re-aplicar al cambiar cualquier filtro
  function guardarYFiltrar() {
    try {
      sessionStorage.setItem(STORAGE_KEY, JSON.stringify({
        sucursal:    document.getElementById('fil-sucursal')?.value || '',
        fecha_desde: document.getElementById('fil-desde')?.value   || '',
        fecha_hasta: document.getElementById('fil-hasta')?.value   || '',
      }));
    } catch(e) {}
    form.submit();
  }

  ['fil-sucursal', 'fil-desde', 'fil-hasta'].forEach(id => {
    document.getElementById(id)?.addEventListener('change', guardarYFiltrar);
  });

  // Limpiar sessionStorage cuando se hace click en "Limpiar"
  document.querySelectorAll('a[href*="cuadratura_diaria_list"]').forEach(a => {
    if (a.href && !a.href.includes('?')) {
      a.addEventListener('click', () => {
        try { sessionStorage.removeItem(STORAGE_KEY); } catch(e) {}
      });
    }
  });
})();
</script>

No modificar la lógica Python de la view. No tocar otros templates.

MEJORA 2 — Volver desde el detalle mantiene el filtro

Esto ya funciona si el usuario usa el botón Atrás del navegador (la URL con 
parámetros GET queda en el historial). No se necesita cambio adicional.
Solo verificar que el enlace "Volver" en templates/cuadratura_diaria/detail.html
use history.back() en vez de un href fijo. Si actualmente tiene un href fijo como:
  <a href="{% url 'control:cuadratura_diaria_list' %}">Volver</a>
Reemplazarlo por:
  <a href="#" onclick="history.back(); return false;">Volver</a>
Si ya usa history.back(), no cambiar nada.

DEPLOY AL NAS

Después de aplicar los cambios, hacer commit y push, luego conectar por SSH al NAS 
y correr actualizar.sh:

git add templates/cuadratura_diaria/list.html templates/cuadratura_diaria/detail.html
git commit -m "feat: auto-filtros y persistencia en lista cuadraturas"
git push origin main

Luego SSH al NAS (192.168.1.24, usuario admin, VPN desactivada) y ejecutar:
cd /volume1/docker/plazagames && bash actualizar.sh

Si actualizar.sh no existe o falla, el proceso manual es:
  git reset --hard origin/main
  docker compose build --no-cache   # o docker-compose según la versión
  docker compose up -d

Verificar que el contenedor "plazagames" quedó corriendo:
  docker ps | grep plazagames
