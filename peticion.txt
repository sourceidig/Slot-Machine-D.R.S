En control/templates/cuadratura_diaria/create.html, en modo edición
(cuadratura.pk existe) la función cargarNumerales() AJAX está
sobreescribiendo los valores guardados de id_numeral_dia,
id_numeral_acumulado e id_caja_anterior con 0, causando que
recalcUI() calcule un descuadre incorrecto.

CAMBIO 1 — Precargar los valores correctos en los inputs al inicio.
Agregar este bloque JS justo antes del cierre del </script> o al
inicio del DOMContentLoaded, solo cuando es modo edición:

  {% if cuadratura.pk %}
  // Modo edición: precargar valores guardados en los inputs de numerales
  document.addEventListener('DOMContentLoaded', function() {
    const ndEl = document.getElementById('id_numeral_dia');
    const naEl = document.getElementById('id_numeral_acumulado');
    const caEl = document.getElementById('id_caja_anterior');
    if (ndEl) ndEl.value = '{{ cuadratura.numeral_dia|default:0 }}';
    if (naEl) naEl.value = '{{ cuadratura.numeral_acumulado|default:0 }}';
    if (caEl) caEl.value = '{{ caja_anterior|default:0 }}';
    // Forzar recalculo con los valores correctos
    if (typeof recalcUI === 'function') recalcUI();
  });
  {% endif %}

CAMBIO 2 — Bloquear cargarNumerales() en modo edición para que
no sobreescriba esos valores. Buscar la función cargarNumerales()
y agregar al inicio:

  async function cargarNumerales() {
    // En modo edición no recalcular numerales desde AJAX
    if (document.getElementById('modo_edicion')) return;
    ... resto de la función ...
  }

Y en el HTML en modo edición agregar el marcador:
  {% if cuadratura.pk %}
  <span id="modo_edicion" style="display:none;"></span>
  {% endif %}
